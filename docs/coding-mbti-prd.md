# 코딩 MBTI 앱 - 제품 기획서

## 📋 프로젝트 개요

### 목적
코딩을 처음 접하는 학생들과 학부모님들이 **"나는 어떤 스타일로 코딩할까?"**를 재미있게 알아보고, 자신에게 맞는 학습 방법을 찾을 수 있도록 돕는 성격 테스트 앱입니다.

마치 MBTI로 자신의 성격을 알아보듯이, 코딩 스타일도 사람마다 다르답니다!

### 누구를 위한 앱인가요?
- 코딩을 한 번도 해보지 않은 초중고 학생
- 코딩 수업을 듣기 시작한 학생
- 코딩에 관심 있는 학생
- 자녀의 코딩 교육을 고민하는 학부모님

---

## 🎯 이 앱으로 무엇을 알 수 있나요?

1. **나를 더 잘 이해하기**: 내가 어떤 방식으로 문제를 푸는지, 어떤 점이 강한지 알 수 있어요
2. **공부 방법 찾기**: 나에게 맞는 공부 방법과 어떤 코딩 언어부터 배우면 좋을지 추천해줘요
3. **팀 프로젝트 잘하기**: 친구들과 함께 할 때 내가 어떤 역할을 맡으면 좋을지 알려줘요
4. **미래 진로 탐색**: 나중에 어떤 개발자가 되면 재미있을지 힌트를 얻을 수 있어요

## 🧬 4가지 코딩 스타일 찾기

### 모두 4가지 질문으로 내 스타일을 찾아요!

#### 1. 문제를 어떻게 풀까요? (생각형 vs 실행형)

**🤔 생각형 (T)**
- 문제를 만나면 먼저 종이에 그림을 그리거나 계획을 세워요
- "왜 그럴까?" "어떻게 하면 더 좋을까?"를 먼저 생각해요
- 수학 문제 풀 때 식을 먼저 세우는 스타일!

**예시**: 레고를 조립할 때 설명서를 먼저 꼼꼼히 읽는 친구

**⚡ 실행형 (P)**
- 일단 해보면서 문제를 풀어요
- 안 되면 다시 시도하고, 점점 나아지게 만들어요
- 수학 문제 풀 때 여러 방법을 시도해보는 스타일!

**예시**: 레고를 일단 만들어보고 다시 고치는 친구

#### 2. 어떻게 정리할까요? (계획형 vs 자유형)

**📋 계획형 (S)**
- 모든 걸 깔끔하게 정리하는 걸 좋아해요
- 공책도 예쁘게 쓰고, 파일도 폴더별로 잘 정리해요
- 미리 계획을 세우고 차근차근 진행해요

**예시**: 방청소할 때 어디부터 정리할지 계획 세우는 친구

**🎨 자유형 (F)**
- 필요할 때마다 방법을 바꿀 수 있어요
- 여러 가지를 동시에 시도하는 걸 좋아해요
- 계획보다는 그때그때 상황에 맞춰요

**예시**: 방청소하면서 생각나는 대로 정리하는 친구

#### 3. 어떻게 배울까요? (혼자형 vs 함께형)

**🎧 혼자형 (I)**
- 혼자 조용히 공부하는 게 편해요
- 책이나 인터넷으로 스스로 찾아보며 배워요
- 집중해서 깊이 파고드는 걸 좋아해요

**예시**: 도서관에서 혼자 공부하는 게 좋은 친구

**👥 함께형 (C)**
- 친구들과 함께 공부하는 게 더 재미있어요
- 모르는 건 물어보고, 아는 건 설명해주며 배워요
- 스터디 그룹이나 동아리 활동을 좋아해요

**예시**: 친구들과 모여서 같이 공부하는 걸 좋아하는 친구

#### 4. 어떤 도구를 좋아할까요? (안정형 vs 신기술형)

**🛡️ 안정형 (E)**
- 많은 사람들이 사용하는 검증된 도구를 선호해요
- 설명이나 도움말이 많은 프로그램이 좋아요
- 안정적이고 믿을 수 있는 걸 중요하게 생각해요

**예시**: 친구들이 많이 쓰는 앱을 사용하는 친구

**🚀 신기술형 (N)**
- 새로 나온 프로그램이나 도구를 시도해보는 걸 좋아해요
- 최신 트렌드와 재미있는 기능에 관심이 많아요
- 남들이 안 써본 걸 먼저 해보는 게 즐거워요

**예시**: 새로 나온 앱을 제일 먼저 깔아보는 친구

## 📝 어떤 질문을 할까요?

### 질문 만드는 방법
- 총 **20~24개** 질문 (각 스타일마다 5~6개씩)
- 둘 중 하나를 선택하거나, 1~5점 중에 고르는 방식
- 코딩을 몰라도 대답할 수 있는 일상적인 상황으로 질문해요
- 정답은 없어요! 자신의 솔직한 생각을 고르면 돼요

### 질문 예시 (쉬운 버전)

#### 스타일 1: 문제 풀이 방법 (생각형 vs 실행형)

**Q1. 레고나 퍼즐을 할 때 나는:**
- A) 먼저 설명서를 읽고 어떻게 만들지 생각한 뒤 시작해요
- B) 일단 조립해보면서 맞춰가요

**Q2. 수학 문제나 퀴즈를 풀 때:**
- A) 왜 그런지 이유를 먼저 생각하고, 공식이나 방법을 찾아요
- B) 여러 방법을 시도해보면서 답을 찾아요

**Q3. 뭔가 안 될 때:**
- A) 처음부터 다시 생각해보고 어디가 잘못됐는지 찾아요
- B) 이것저것 바꿔가며 될 때까지 계속 시도해요

#### 스타일 2: 정리 방법 (계획형 vs 자유형)

**Q4. 새로운 프로젝트나 숙제를 시작할 때:**
- A) 먼저 어떻게 할지 계획을 세우고, 순서대로 진행해요
- B) 일단 시작해보고, 하다가 필요한 부분을 채워나가요

**Q5. 공부한 내용이나 파일을 정리할 때:**
- A) 폴더를 만들고 이름을 정해서 깔끔하게 정리해요
- B) 필요할 때 찾기만 하면 되니까, 크게 신경 쓰지 않아요

**Q6. 친구들과 발표 준비를 할 때:**
- A) 누가 무엇을 할지 역할을 정하고, 계획표를 만들어요
- B) 하면서 필요한 걸 나눠서 하고, 유연하게 조정해요

#### 스타일 3: 공부 방법 (혼자형 vs 함께형)

**Q7. 새로운 걸 배울 때 나는:**
- A) 책이나 유튜브 영상을 보며 혼자 공부하는 게 좋아요
- B) 친구들이나 선생님과 함께 배우는 게 더 재미있어요

**Q8. 모르는 문제가 생기면:**
- A) 인터넷으로 검색하거나 책을 찾아보며 스스로 해결해요
- B) 아는 사람에게 물어보고 함께 이야기하며 해결해요

**Q9. 공부할 때 가장 집중이 잘 되는 환경은:**
- A) 조용한 방이나 도서관에서 혼자 있을 때
- B) 친구들과 함께 있거나 이야기 나누면서 할 때

#### 스타일 4: 도구 선택 (안정형 vs 신기술형)

**Q10. 새로운 앱이나 게임을 선택할 때:**
- A) 많은 사람들이 쓰고 추천이 많은 걸 골라요
- B) 새로 나왔거나 재미있어 보이는 걸 먼저 써봐요

**Q11. 어떤 프로그램이나 도구가 마음에 드나요?**
- A) 설명이 많고, 잘 작동하는 검증된 프로그램
- B) 최신 기능이 있고, 새롭고 신기한 프로그램

**Q12. 유튜브나 블로그에서 관심 가는 내용:**
- A) 많은 사람들이 쓰는 유용한 꿀팁
- B) 새로 나온 것들이나 트렌드

## 🎨 결과 화면에서 보여줄 내용

### 테스트 결과로 뭘 알려줄까요?

1. **내 유형과 별명**
   - 예: TSIE - "튼튼한 설계자"
   - 재미있는 별명으로 기억하기 쉽게!

2. **내 성향 그래프**
   - 거미줄 모양 그래프로 4가지 스타일을 한눈에!
   - 각 스타일이 어떤 의미인지 쉽게 설명

3. **내가 잘하는 것 & 연습하면 좋은 것**
   - 강점: "계획을 잘 세우고 체계적으로 일해요!"
   - 연습하면 좋은 것: "빠르게 만들어보는 연습을 해보세요!"

4. **나에게 맞는 공부 방법**
   - 어떤 코딩 언어부터 배우면 좋을지
   - 어떤 공부 방법이 나에게 맞는지
   - 해보면 좋을 재미있는 프로젝트

5. **미래에 이런 일을 할 수 있어요**
   - 서버 개발자, 앱 개발자, 게임 개발자 등
   - 쉽게 이해할 수 있는 직업 설명

6. **친구들과 함께 일할 때 팁**
   - 내가 어떤 역할을 하면 좋은지
   - 다른 친구들과 어떻게 협력하면 좋은지

7. **유명한 사람과 비교하기**
   - "당신은 마크 저커버그(페이스북 만든 사람)와 비슷해요!"

8. **친구들과 공유하기**
   - 예쁜 결과 이미지 만들어서 친구들한테 자랑하기
   - 친구들 결과와 비교해보기

## 🏗️ 어떻게 만들까요? (개발자용)

### 화면 개발 (프론트엔드)
- **기본 도구**: React (리액트) + TypeScript (타입스크립트)
- **디자인**: Tailwind CSS (예쁜 화면 만들기)
- **그래프**: Recharts (거미줄 그래프 그리기)
- **애니메이션**: Framer Motion (움직이는 효과)

### 서버 개발 (선택사항)
- **서버 도구**: Next.js (간단한 서버)
- **데이터 저장**: 나중에 결과를 다시 볼 수 있게
- **로그인**: 필요하면 나중에 추가

### 인터넷에 올리기
- **Vercel** 또는 **Netlify** (무료로 쉽게 배포!)

## 📅 만드는 순서

### 1단계: 준비하기 (1주일)
- [ ] 질문 20~24개 완성하기
- [ ] 점수 계산하는 방법 정하기
- [ ] 16가지 유형별 설명 작성하기
- [ ] 화면 디자인 그려보기

### 2단계: 기본 기능 만들기 (2-3주일)
- [ ] 프로젝트 시작하기
- [ ] 질문 화면 만들기
- [ ] 선택한 답으로 점수 계산하기
- [ ] 결과 화면 만들기
- [ ] 휴대폰에서도 잘 보이게 만들기

### 3단계: 결과 화면 예쁘게 (1-2주일)
- [ ] 거미줄 그래프 넣기
- [ ] 설명 내용 채우기
- [ ] 추천 내용 보여주기
- [ ] 움직이는 효과 넣기

### 4단계: 더 재미있게 (1-2주일)
- [ ] 결과 이미지 만들기 기능
- [ ] SNS 공유 버튼
- [ ] 결과 저장하고 다시 보기
- [ ] 통계 보여주기 (선택)

### 5단계: 테스트하고 완성 (1주일)
- [ ] 친구들한테 써보라고 하기
- [ ] 고칠 점 수정하기
- [ ] 인터넷 검색에 잘 나오게 하기
- [ ] 인터넷에 올리기

## 🎯 우리 앱만의 특별한 점

1. **초보자도 쉽게**: 코딩을 몰라도 재미있게 할 수 있어요
